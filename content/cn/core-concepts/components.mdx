文档组件用于表达语义，例如提示、对比、结构块与列布局。多数组件既可用 MDX 标签，也支持 Markdown 指令。

## Callout

Callout 用于提示类信息，建议按语义选择不同级别：

- `Tip/Note`：白色提示块，适合轻量提醒或补充说明。
- `Info`：蓝色提示块，适合稳定信息或一致性说明。
- `Warning`：橙色提示块，适合风险提示或强约束。
- `Deprecated`：红色提示块，适合弃用说明与迁移提醒。

<Tip title="Tip">
用于补充说明或小技巧提示。
</Tip>

<Info title="Info">

## Alerts（GitHub 语法）

使用 GitHub 风格的提示语法，会自动映射为 Callout。

> 会渲染成 Note 提示块，便于迁移和协作时统一语法。

> [!WARNING]
> 用于风险提示或强约束说明。

## Math Equations

行内公式用 `$...$`，块级公式用 `$$...$$`，渲染由 KaTeX 提供。

行内示例：$E = mc^2$。

$$
\int_0^1 x^2 \, dx = \frac{1}{3}
$$

## Diff Block

用于并排对比两个方案/版本，`beforeLabel` 与 `afterLabel` 可选，必要时再显示顶部标签。

<DiffBlock beforeLabel="旧行为" afterLabel="新行为">
  <DiffColumn>
    <code>cache: "manual"</code>
    <p>需要手动维护缓存策略。</p>
  </DiffColumn>
  <DiffColumn highlight>
    <code>cache: "filesystem"</code>
    <p>自动与文件结构同步。</p>
  </DiffColumn>
</DiffBlock>

## Code Block

使用标准 Markdown 代码块语法，渲染时自动应用统一样式与复制按钮。

```ts
export const navStrategy = "filesystem";
export const contentPath = "content";
```

## Colored Diffs

代码块中以 `+` / `-` 开头的行会自动渲染为增删差异背景。

```diff
- cache: "manual"
+ cache: "filesystem"
```

## Code Tabs

用于在同一块内切换多段代码。使用 `code-group` 指令即可，无需写额外 JSX。

::: code-group
```javascript [Chat Completions] subtitle="Structured Outputs"
const completion = await openai.chat.completions.create({
  model: "gpt-5",
  messages: [{ role: "user", content: "Jane, 54 years old" }],
});
```

```javascript [Responses] subtitle="Structured Outputs"
const response = await openai.responses.create({
  model: "gpt-5",
  input: "Jane, 54 years old",
});
```
:::

## Steps

将有序列表渲染为步骤块，适合安装/迁移/排查流程。

::: steps
1. 安装 `typematter` 依赖。
2. 调整 `content/` 目录结构。
3. 构建前先执行 `npm run validate:docs`。
:::

## Details / Accordion

折叠块用于放置补充说明或可选项，避免打断阅读主线。

::: details 为什么重要
保持长说明的可读性，同时不干扰主流程。
:::
