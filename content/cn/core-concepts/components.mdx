文档组件用于表达语义，例如提示、对比、结构块与列布局。多数组件既可用 MDX 标签，也支持 Markdown 指令。

如果你主要关心写法速查，而不是视觉效果，直接看[指令速查表](/cn/core-concepts/authoring-syntax)。

## Callout

Callout 用于提示类信息，建议按语义选择不同级别：

- `Tip/Note`：白色提示块，适合轻量提醒或补充说明。
- `Info`：蓝色提示块，适合稳定信息或一致性说明。
- `Warning`：橙色提示块，适合风险提示或强约束。
- `Deprecated`：红色提示块，适合弃用说明与迁移提醒。

<Tip title="Tip">
用于补充说明或小技巧提示。
</Tip>

<Info title="Info">
用于稳定的规则或信息提示。
</Info>

<Warning title="Warning">
用于风险提示或强约束。
</Warning>

<Deprecated title="Deprecated">
用于标记弃用功能或迁移提醒。
</Deprecated>

## Math Equations

行内公式用 `$...$`，块级公式用 `$$...$$`，渲染由 KaTeX 提供。

行内示例：$E = mc^2$。

$$
\int_0^1 x^2 \, dx = \frac{1}{3}
$$

## Diff Block

用于并排对比两个方案/版本，`beforeLabel` 与 `afterLabel` 可选，必要时再显示顶部标签。

<DiffBlock beforeLabel="旧行为" afterLabel="新行为">
  <DiffColumn>
    <code>cache: "manual"</code>
    <p>需要手动维护缓存策略。</p>
  </DiffColumn>
  <DiffColumn highlight>
    <code>cache: "filesystem"</code>
    <p>自动与文件结构同步。</p>
  </DiffColumn>
</DiffBlock>

## Code Block

使用标准 Markdown 代码块语法，渲染时自动应用统一样式与复制按钮。

```ts
export const navStrategy = "filesystem";
export const contentPath = "content";
```

## Colored Diffs

代码块中以 `+` / `-` 开头的行会自动渲染为增删差异背景。

```diff
- cache: "manual"
+ cache: "filesystem"
```

## Code Tabs

用于在同一块内切换多段代码。使用 `code-group` 指令即可，无需写额外 JSX。

::: code-group
```javascript [Chat Completions] subtitle="Structured Outputs"
const completion = await openai.chat.completions.create({
  model: "gpt-5",
  messages: [{ role: "user", content: "Jane, 54 years old" }],
});
```

```javascript [Responses] subtitle="Structured Outputs"
const response = await openai.responses.create({
  model: "gpt-5",
  input: "Jane, 54 years old",
});
```
:::

## Endpoint / ParamTable / ResponseSchema

这组组件适合 API 页面，重点是把接口头、参数、返回结构拆成稳定语义块，而不是堆散乱表格。

<Endpoint
  method="GET"
  path="/v1/docs/[slug]"
  title="获取文档页面"
  summary="按路由返回 registry 中的页面元信息。"
  auth="optional"
  since="0.3.0-rc"
>
  <ParamTable title="Path params">
    <ParamField name="slug" type="string[]" required>
      文档路由片段数组。
    </ParamField>
    <ParamField name="lang" type='"cn" | "en"' defaultValue="cn">
      语言代码。
    </ParamField>
  </ParamTable>

  <ResponseSchema title="200 Response" code="200" mediaType="application/json">
    <SchemaField name="title" type="string" required description="页面标题" />
    <SchemaField name="section" type="string" required description="所属分组" />
    <SchemaField name="toc" type="TocItem[]" description="标题目录">
      <SchemaField name="id" type="string" required description="锚点 id" />
      <SchemaField name="title" type="string" required description="标题文本" />
      <SchemaField name="level" type="number" required description="标题层级" />
    </SchemaField>
  </ResponseSchema>
</Endpoint>

### 指令语法

~~~mdx
:::endpoint{method="GET" path="/v1/docs/[slug]" title="获取文档页面" summary="按路由返回 registry 中的页面元信息。" auth="optional" since="0.3.0-rc"}
:::param-table[Path params]
::param-field[slug]{type="string[]" required="true" description="文档路由片段数组。"}
::param-field[lang]{type='"cn" | "en"' default="cn" description="语言代码。"}
:::

:::response-schema[200 Response]{code="200" mediaType="application/json"}
::schema-field[title]{type="string" required="true" description="页面标题"}
::schema-field[section]{type="string" required="true" description="所属分组"}
:::schema-field[toc]{type="TocItem[]" description="标题目录"}
::schema-field[id]{type="string" required="true" description="锚点 id"}
::schema-field[title]{type="string" required="true" description="标题文本"}
::schema-field[level]{type="number" required="true" description="标题层级"}
:::
:::
:::
~~~

## DoDont

用于最佳实践、迁移规范、代码 review 规则这类非常高频的“该做 / 不该做”表达。

<DoDont title="写 API 文档时">
  <DoItem title="保持字段名稳定">
    对外字段名、枚举值和错误码应该保持长期稳定。
  </DoItem>
  <DoItem title="写清错误分支">
    除了 happy path，也要明确 4xx / 5xx 的返回形态。
  </DoItem>
  <DontItem title="不要把 breaking change 埋在正文里">
    版本破坏性变化应有显式标记，而不是只在长段落里顺带提到。
  </DontItem>
  <DontItem title="不要只给示例不给结构">
    示例响应不能替代字段级结构定义。
  </DontItem>
</DoDont>

### 指令语法

~~~mdx
:::do-dont[写 API 文档时]
:::do[保持字段名稳定]
对外字段名、枚举值和错误码应该保持长期稳定。
:::

:::dont[不要只给示例不给结构]
示例响应不能替代字段级结构定义。
:::
:::
~~~

## VersionGate

用于明确能力从何时可用、何时弃用、何时移除。

<VersionGate
  title="Legacy search index"
  since="0.2.0"
  deprecated="0.4.0"
  removedIn="1.0.0"
  replacement="standard search manifest"
>
  当一个能力进入迁移期时，不要只写一句“未来会调整”，直接把版本策略贴在能力块附近。
</VersionGate>

### 指令语法

~~~mdx
:::version-gate[Legacy search index]{since="0.2.0" deprecated="0.4.0" removedIn="1.0.0" replacement="standard search manifest"}
当一个能力进入迁移期时，不要只写一句“未来会调整”，直接把版本策略贴在能力块附近。
:::
~~~
