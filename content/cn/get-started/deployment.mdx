Typematter 使用 **Next.js 静态导出**。`npm run build` 会先校验文档，再生成 registry、搜索资产和可部署站点，最终产物位于 `out/`。

## 推荐仓库模型

把官方 Typematter 仓库当作维护中的 demo 和参考实现。正式团队或产品文档应生成独立 starter 仓库。

```bash
# 在官方 Typematter 仓库根目录执行。
npm install
npm run typematter -- init --dir ../my-docs
cd ../my-docs
```

<Callout type="info" title="为什么要独立仓库">
把业务文档放在官方仓库外，可以避免上游示例内容和自己的内容、配置、发布流程混在一起。
</Callout>

## 生产环境变量

复制 starter 环境变量文件：

```bash
cp .env.example .env.local
```

至少设置：

```bash
TYPEMATTER_SITE_URL=https://docs.example.com
NEXT_PUBLIC_SITE_URL=https://docs.example.com
```

`TYPEMATTER_SITE_URL` 对生产环境是必需的，因为它会写入 `robots.txt` 和 `sitemap.xml`。如果缺失，生成文件会退回到示例域名。

可选 Ask AI 变量：

```bash
NEXT_PUBLIC_TYPEMATTER_ASK_AI_ENDPOINT=https://your-worker-domain.workers.dev
NEXT_PUBLIC_TYPEMATTER_ASK_AI_TIMEOUT_MS=25000
# NEXT_PUBLIC_TYPEMATTER_ASK_AI_ENABLED=false
```

配置 endpoint 后 Ask AI 会启用。只有需要强制关闭时，才设置 `NEXT_PUBLIC_TYPEMATTER_ASK_AI_ENABLED=false`。

## 构建与预览

:::steps
1. 运行 `npm run validate:docs`。
2. 运行 `npm run build`。
3. 用 `npm start` 预览静态产物。
4. 部署 `out/` 目录。
:::

## 构建产物

导出前，Typematter 会在项目内写入生成资产：

- `.typematter/registry.json`
- `public/typematter/search/`
- `public/typematter/ask-index.json`
- `public/robots.txt`
- `public/sitemap.xml`

导出后，可部署副本位于 `out/`：

- `out/`
- `out/typematter/search/`
- `out/typematter/ask-index.json`
- `out/robots.txt`
- `out/sitemap.xml`

不要手动修改生成资产。需要变更时，修改 MDX、配置或插件后重新构建。

## 托管目标

<Cards columns="3" mobileColumns="1">
  <Card title="Vercel" description="使用静态导出的 out 目录发布。" />
  <Card title="Cloudflare Pages" description="推送后自动构建并全球分发静态产物。" />
  <Card title="Nginx / S3 / R2 / OSS" description="把 out 当作普通静态文件上传。" />
</Cards>

### Vercel

- Build Command: `npm run build`
- Output Directory: `out`
- Environment Variables: `TYPEMATTER_SITE_URL`、`NEXT_PUBLIC_SITE_URL`、可选 Ask AI 变量

### Cloudflare Pages

- Build Command: `npm run build`
- Build Output Directory: `out`
- Environment Variables: 同上
- Ask AI Worker: 如需启用，单独部署 `integrations/cloudflare-ask-ai-worker/`

### Nginx / S3 / R2 / OSS

- 在本地或 CI 运行 `npm run build`。
- 上传 `out/` 的内容。
- 确保托管服务能正常分发 HTML、CSS、JS、JSON、XML 和 TXT 文件。

## Ask AI 是可选能力

无论是否启用 Ask AI，文档站本身都是完整静态站点。只有希望搜索弹窗出现 AI 回答页签时，才需要部署 Worker。

参考实现：

- `integrations/cloudflare-ask-ai-worker/README.md`
- `integrations/cloudflare-ask-ai-worker/src/index.ts`
- `integrations/cloudflare-ask-ai-worker/wrangler.toml`

## 常见问题

### `robots.txt` 或 `sitemap.xml` 仍指向 `example.com`

在运行 `npm run build` 前设置 `TYPEMATTER_SITE_URL`。

### 构建因孤儿页面或断链失败

运行 `npm run validate:docs`，按报告修正文档、导航或链接问题。

### 我需要不包含官方内容的干净项目

在官方仓库根目录运行 `npm run typematter -- init --dir ../my-docs`。目标目录必须位于官方仓库外部。
