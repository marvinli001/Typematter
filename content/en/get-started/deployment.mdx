Typematter deploys as a **Next.js static export**. `npm run build` validates the docs, builds the registry and search assets, and writes the deployable site to `out/`.

## Recommended repository model

Use the official Typematter repository as the maintained demo and reference implementation. For product or team docs, generate a separate starter repository.

```bash
# Run from the official Typematter repository root.
npm install
npm run typematter -- init --dir ../my-docs
cd ../my-docs
```

<Callout type="info" title="Why separate repositories">
Keeping your product docs outside the official repo avoids mixing upstream examples with your own content, config, and release workflow.
</Callout>

## Production environment

Copy the starter environment file:

```bash
cp .env.example .env.local
```

Set at least:

```bash
TYPEMATTER_SITE_URL=https://docs.example.com
NEXT_PUBLIC_SITE_URL=https://docs.example.com
```

`TYPEMATTER_SITE_URL` is required for production correctness because it is used in `robots.txt` and `sitemap.xml`. If it is omitted, generated files fall back to the example domain.

Optional Ask AI variables:

```bash
NEXT_PUBLIC_TYPEMATTER_ASK_AI_ENDPOINT=https://your-worker-domain.workers.dev
NEXT_PUBLIC_TYPEMATTER_ASK_AI_TIMEOUT_MS=25000
# NEXT_PUBLIC_TYPEMATTER_ASK_AI_ENABLED=false
```

An endpoint enables Ask AI. Set `NEXT_PUBLIC_TYPEMATTER_ASK_AI_ENABLED=false` only when you need to force it off.

## Build and preview

:::steps
1. Run `npm run validate:docs`.
2. Run `npm run build`.
3. Preview the static output with `npm start`.
4. Deploy the `out/` directory.
:::

## Build outputs

Before export, Typematter writes generated assets into the project:

- `.typematter/registry.json`
- `public/typematter/search/`
- `public/typematter/ask-index.json`
- `public/robots.txt`
- `public/sitemap.xml`

After export, deployable copies live under `out/`:

- `out/`
- `out/typematter/search/`
- `out/typematter/ask-index.json`
- `out/robots.txt`
- `out/sitemap.xml`

Do not hand-edit generated assets. Change MDX content, config, or plugins, then rebuild.

## Hosting targets

<Cards columns="3" mobileColumns="1">
  <Card title="Vercel" description="Use static export output with the out directory." />
  <Card title="Cloudflare Pages" description="Build on push and publish static output globally." />
  <Card title="Nginx / S3 / R2 / OSS" description="Upload out as plain static files." />
</Cards>

### Vercel

- Build Command: `npm run build`
- Output Directory: `out`
- Environment Variables: `TYPEMATTER_SITE_URL`, `NEXT_PUBLIC_SITE_URL`, optional Ask AI variables

### Cloudflare Pages

- Build Command: `npm run build`
- Build Output Directory: `out`
- Environment Variables: same as above
- Ask AI Worker: deploy `integrations/cloudflare-ask-ai-worker/` separately if needed

### Nginx / S3 / R2 / OSS

- Run `npm run build` locally or in CI.
- Upload the contents of `out/`.
- Configure your host to serve static HTML, CSS, JS, JSON, XML, and TXT files.

## Ask AI remains optional

The docs site is fully static with or without Ask AI. The Worker is only needed when the search modal should include an AI answer tab.

Reference implementation:

- `integrations/cloudflare-ask-ai-worker/README.md`
- `integrations/cloudflare-ask-ai-worker/src/index.ts`
- `integrations/cloudflare-ask-ai-worker/wrangler.toml`

## Troubleshooting

### `robots.txt` or `sitemap.xml` still points to `example.com`

Set `TYPEMATTER_SITE_URL` before running `npm run build`.

### Build fails on orphan pages or broken links

Run `npm run validate:docs` and fix the reported page, nav, or link issue.

### I want a clean project without the official docs content

Run `npm run typematter -- init --dir ../my-docs` from the official repository root. The target directory must be outside the official repo.
