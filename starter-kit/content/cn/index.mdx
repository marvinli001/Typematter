这是一个干净的 Typematter starter。把这里替换成你自己的产品文档即可。

## 这个 starter 提供什么

- 完整的静态文档壳
- 双语目录结构示例
- 搜索、目录、分页和组件样式
- 构建期文档校验

<Note title="内容目录">
你的实际文档只需要维护在 `content/` 里，其余代码通常只在改样式或扩展组件时才需要动。
</Note>

## 下一步

::: steps
1. 修改 `site.config.ts` 的站点标题和域名。
2. 按你的信息架构重写 `content/` 目录。
3. 本地执行 `npm run validate:docs` 和 `npm run build`。
:::
