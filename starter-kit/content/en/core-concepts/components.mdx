Typematter works best when recurring documentation patterns become stable semantic components instead of ad-hoc markdown styling.

## Built in

- Callout
- Steps
- Cards
- Details
- FileTree
- CodeTabs
- DiffBlock

<Info title="Recommendation">
Start with a small set of high-frequency components, then extend for real content needs. Avoid large one-off component inventories.
</Info>
