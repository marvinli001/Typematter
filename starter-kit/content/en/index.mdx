This is a clean Typematter starter. Replace the sample pages with your own docs.

## What the starter includes

- Static docs shell
- Bilingual content structure
- Search, TOC, pager, and MDX component styles
- Build-time docs validation

<Note title="Content first">
Your real product docs should live under `content/`. Most code changes happen only when you want to restyle the shell or add new semantic components.
</Note>

## Next steps

::: steps
1. Update `site.config.ts` with your title and domain.
2. Rewrite the `content/` tree around your information architecture.
3. Run `npm run validate:docs` and `npm run build`.
:::
