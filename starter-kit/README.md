# My Docs

Typematter starter project.

## Development

```bash
npm install
npm run dev
```

## Build

```bash
npm run build
```

## Validate docs

```bash
npm run validate:docs
```

## Configure

1. Edit `site.config.ts` and `nav.config.ts`.
2. Replace the sample pages under `content/`.
3. Set `TYPEMATTER_SITE_URL` before production builds so `robots.txt` and `sitemap.xml` are generated correctly.

## Starter workflow

- The official Typematter repository is the maintained docs/demo site.
- This starter is the clean project scaffold generated by `typematter init`.
- Keep your own docs in a separate Git repository so your product content and the Typematter upstream can evolve independently.
